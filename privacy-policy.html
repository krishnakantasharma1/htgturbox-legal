📊 Information We Collect

1. Device Information (Processed Locally — Never Transmitted)

🔧 App Permissions Explained

HTG TurboX requests only the permissions necessary for its features. Below is a complete explanation of each permission:

Standard Permissions

  • INTERNET — Required for the Network Check feature (ping tests to google.com, cloudflare.com, 8.8.8.8). No personal data is transmitted.
  • ACCESS_NETWORK_STATE — Checks whether the device has an active connection before running Network Check.
  • ACCESS_NOTIFICATION_POLICY — Enables the Do Not Disturb (DND) feature in Game Booster and Game Toolbox. DND is only activated when the user explicitly enables it.
  • POST_NOTIFICATIONS (Android 13+) — Sends periodic informational reminders about device stats. Can be disabled anytime via system settings.
  • RECEIVE_BOOT_COMPLETED — Reschedules notification reminders after device restart via Android WorkManager.
  • REQUEST_DELETE_PACKAGES — Used by the App Manager to open Android's standard uninstall confirmation dialog.
  • QUERY_ALL_PACKAGES — Required on Android 11+ to discover installed apps for the Game Launcher and App Manager.

System Settings Permissions

  • WRITE_SETTINGS — Used by Game Booster to toggle rotation lock (ACCELEROMETER_ROTATION system setting). Requires the user to manually enable in system settings.
  • SYSTEM_ALERT_WINDOW — Used by the Game Toolbox floating overlay to display stats, toggles, and the Boost button over games. Requires the user to manually enable "Display over other apps" in system settings. The overlay only starts when the user launches a game and has the feature toggled on.

Service Permissions

  • FOREGROUND_SERVICE — Required to run the Game Toolbox overlay as a foreground service with a persistent notification.
  • FOREGROUND_SERVICE_SPECIAL_USE (Android 14+) — Required to start the overlay foreground service on Android 14 and above. The service is declared with foregroundServiceType="specialUse" for the game overlay toolbox.

ℹ️ HTG TurboX does not request any permissions beyond those listed above. Every permission is tied to a specific user-facing feature and is requested at runtime with clear explanation.

🎮 Game Toolbox Overlay

The Game Toolbox is an optional floating overlay that displays live stats (RAM, battery, temperature, FPS) and provides quick toggles (DND, rotation lock, FPS counter) while gaming.

  • The overlay requires SYSTEM_ALERT_WINDOW permission which the user must manually grant in system settings
  • The overlay only starts when the user launches a game through the App with the Game Toolbox toggle enabled
  • The overlay runs as a foreground service (GameToolboxService) with a persistent notification showing a "Stop" button
  • The overlay can be closed by dragging the floating icon to the ✕ close zone, tapping "Stop" in the notification, or disabling the toggle in the App
  • The overlay does not collect, store, or transmit any data. All stats are read from standard Android APIs and displayed in real-time
  • The "Boost" button clears only the App's own cache (cacheDir) and requests garbage collection (System.gc()). It does not modify any other app's data

🗂️ Storage Analyzer

The Storage Analyzer helps users understand their device storage usage:

  • Displays total, used, and free storage space using Android's StatFs API — no special permissions required
  • Shows installed apps sorted by size using StorageStatsManager — only reads app metadata (name, package, size), not app contents
  • Provides an "Info" button for each app that opens Android's standard App Info settings page
  • Links to Android Settings → Storage for detailed system storage management
  • No files are accessed, scanned, uploaded, or transmitted to any server
  • The Storage Analyzer does not access photos, videos, audio files, or any media on your device

🔑 Shizuku Integration (Optional Advanced Tweaks)

HTG TurboX optionally integrates with Shizuku, a third-party open-source tool that enables ADB-level commands without root access. Shizuku is developed by Rikka (rikka.app) and is available on Google Play Store.

How it works:

  • Shizuku is NOT bundled with HTG TurboX — the user must install it separately from Google Play Store
  • Shizuku must be started manually by the user via Wireless Debugging
  • HTG TurboX requests Shizuku permission via a standard permission dialog — the user must explicitly grant access
  • All Shizuku features are entirely optional — the App functions fully without Shizuku

What Shizuku-powered features do:

  • Animation Speed — modifies window_animation_scale, transition_animation_scale, animator_duration_scale via "settings put global"
  • Background Process Limit — modifies background_process_limit via "settings put global"
  • Max Cached Processes — modifies activity_manager_constants via "settings put global"
  • Performance Mode — sets power mode via "cmd power set-mode"
  • Force GPU Rendering — modifies force_gpu_rendering via "settings put global"
  • App Hibernation — force-stops user-selected apps via "am force-stop"
  • Screen Density (DPI) — modifies display density via "wm density"
  • Disable Bloatware — disables user-selected system apps via "pm disable-user" (fully reversible)
  • WiFi Scan Throttling — modifies wifi_scan_always_enabled via "settings put global"
  • Disable HW Overlay — modifies disable_overlay_optimization via "settings put global"
  • Refresh Rate Control — modifies peak_refresh_rate via "settings put system"
  • Clear All App Caches — runs "pm trim-caches" which is the same API Android Settings uses internally. Only clears cached data, never app data or user settings

What Shizuku features do NOT do:

  • Do not access, read, or modify any app's private data, files, or databases
  • Do not install or uninstall any app (pm install/uninstall is blocked)
  • Do not modify system partitions or kernel parameters
  • Do not access root (su) in any way
  • Do not transmit any data to any server

All changes made by Shizuku features modify Android's system settings database, which persists after reboot. Users can undo all changes using the "Reset All Advanced Tweaks" button or by manually changing settings in Developer Options.

🔒 Shizuku integration is fully optional. Users who do not install or enable Shizuku can use all other App features without any limitation.

⚡ Module Flasher

The Module Flasher allows users to load and execute .turbox module files — JSON configuration files containing predefined sets of system tweaks.

  • Modules are JSON files that contain only whitelisted command types — no arbitrary shell execution is possible
  • All module commands are validated and sanitized at parse time before execution
  • Dangerous patterns (rm, su, chmod, mount, reboot, curl, etc.) are explicitly blocked at multiple validation layers
  • Module files are selected by the user from their device storage via Android's standard file picker
  • The App does not download, distribute, or auto-execute any modules — the user must manually select and confirm each flash
  • Module execution requires Shizuku to be installed, running, and granted permission
  • All module actions are the same whitelisted commands available in the Advanced Tweaks section
